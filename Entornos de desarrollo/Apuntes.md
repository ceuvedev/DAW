# SCRUM master

Metodologías de desarrolo agil.

Nace de la necesidad de atender de forma rápida a los constantes cambios de un proyecto.

Una de las más famosas es SCRUM, que ofrece un marco de trabajo para la gestion de proyectos de manera ágil.

El exito de esta es asumir que el proyecto no está todo definido y que puede cambiar el algún momento dado y adaptarse.

El SCRUM funciona dividiendo el proyecto en pequeñas tareas, cada una con un objetivo concreto y que se realizará en un espacio/tiempo asumible. Este espacio/tiempo se llama sprint y suele ser dos semanas, como mucho un mes.

En este marco es de vital importancia la comunicación y la colaboración de los miembros del equipo, de igual manera con el cliente..

El SCRUM es útil para un determinado contexto, es ideal para equipos pequeños, aproximadamente 10 personas.

Los roles  del equipo son:
    - SCRUM Master, es el miembro que se encarga de que el framework se aplica correctamente. Es quién dirige las reuniones, las organiza y protege al equipo de desarrollo de las distracciones. No equivocarse con el team leader o gestor del proyecto. el SCRUM MASTER es una guía del proceso, no indica qué ni cómo hay que hacerlo.

    - Product Owner, se trata de un representante del cliente. él define las necesidades del producto, las prioriza y asegura que el resultado del equipo son adecuados desde el punto de vista del negocio. es más un rol comercial, que no intefiere en los aspectos técnicos y con el equipo de desarrollo sirve de enlace, entre éste y el cliente.

    - Equipo de desarrollo, es el responsable de entregar las diferentes partes del producto en cada sprint. así como el producto final. 

Aunque el SCRUM asume que los requisitos de un proyecto pueden variar durante su desarrollo, es necesario siempre plantear unos objetivos. para poder dividirlos en pequeñas piezas para realizar.

Nuestro punto de partida es una lista de tareas ya ordenadas según la prioridad, conocido como Product backlog. Esta lista está generada por el product owner atendienddo a los objetivos iniciales planteadas por el cliente.

Los planteamientos que se pueden realizar en las reuniones de SCRUM son:

    - Reunión de planificación del sprint (sprint planning) se realiza antes de comenzar el sprint. en ella se plantean las tareas y objetivos de esta, a partir de las tareas más prioritarias del product backlog, y así como el tiempo necesario para entregar el producte al final del sprint. a la lista de tareas del sprint sprint backlog, el tiempo sugerido por el SCRUM para esta reunión son 4 horas.

    - Daily SCRUM, se realiza generalmente al comienzo de cada jornada laboral, a una hora determinada. en ellos se sincroniza las tareas del equipo, comentando el estado en que se encuentra el trabajo
