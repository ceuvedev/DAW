## PROJECTE TIC

Analisis de requisitos en un proyecto TIC es la primera fase del proyecto intermodular. El objetivo es entender la problematica que se quiere resolver.

Hay que documentar lo importante:
 - las necesidades
 - funcionalidades
 - condiciones

### Objetivos

- Identificar las necesidades reales de los usuarios y el cliente
- Definir de manera precisa, priorizada y verificacle el requisitos del proyecte.
- Diferencias entre lo que se desea (requisitos) y cómo se hará (diseño)
  
### Aspectos implicados en el analisis de requisitos

Tipos de requisitos:
    Requisitos funcionesles
        Describen qué tiene que hacer el sistema
    Requisitos no funcionales
        Describen como se ha de comportar el sistema
    Requisitos de dominio o normativos
        Restricciones derivadas del entorno legal, normativa o de negocio.
    Restricciones técnicas
        Limitaciones relacionadas con tecnologías, plataformas o recursos.
Actores y usuarios
    ACtor: Cualquier persona, sistema o dispositivo que interactua con la aplicación.
    Usuario final: persona que hará uso directo de la aplicación
    Parte interesada (stakeholder): cualquier agente implicado (cliente, desarrolladores, administradores, profesorado)
Lo que abarca el producto y la visión de la misma
    Define qué incluye o qué no dentro del proyecto
    Se resume en un documento de visió corto que contiene:
        Problema para resolver
        Objetivos del sistema
        Usuarios i actors principals
        Beneficios esperados
        Limitaciones o restricciones iniciales
