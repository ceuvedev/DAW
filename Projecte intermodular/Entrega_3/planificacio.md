### Análisis inicial del proyecto

#### Definición del projecto

- **Objetivo principal:** El objetivo del proyecto es desarrollar una aplicación web para la gestión de citas en una óptica, con el fin de proporcionar herramientas adaptadas a las nuevas tecnologías. De esta forma, se busca mejorar la productividad del profesional mediante una mejor organización y gestión de las citas.

- **Usuarios o clientes objetivo:** El target principal del gestor de citas serán los pacientes y el personal de la óptica (administradores).

- **Funcionalidades esenciales:** Agendar citas, revisión de la ficha optométrica y visualización de la agenda en tiempo real.

- **Tecnologías previstas:** 
  - **Frontend:** HTML, CSS, JS
  - **Backend:** Node.js (Express)
  - **Base de datos:** MongoDB
  - **Servidor:** Nginx
  - **Herramientas:** Git, Docker, Figma
  - **Requisitos iniciales:** Al tener la opción de guardar fichas, por ley hay que conservar los datos del cliente durante 5 años.

- **Tipo de aplicación:** Se trata de una aplicación web basada en arquitectura cliente-servidor, con comunicación mediante API REST.

#### Restricciones y recursos

Factores que pueden condicionar el desarrollo del proyecto.

- **Recursos humanos disponibles:** El equipo es pequeño, por lo que es posible que el desarrollo del proyecto se alargue.
- **Recursos tecnológicos:** Al encontrarnos en una fase inicial del ciclo, hay módulos en los que todavía no se dispone de todos los conocimientos necesarios.
- **Limitaciones técnicas:** Es necesario cumplir con requisitos de seguridad, ya que se trabaja con datos sensibles de pacientes.
- **Compatibilidad:** La aplicación debe ser compatible con los principales navegadores web.
- **Accesibilidad:** La interfaz debe ser clara y fácil de usar para todo tipo de usuarios.
